Version control system
1. 記錄開發到成品的過程
2. 確保不同人或是團隊編輯同一程式檔案都能獲得同步


Why we need it ?
1. 多人協作時,  檔案不會被蓋掉
2. 保留歷史訊息, 以供查詢
3. 隨時復原修改, 可以回到之前版本
4. 開發過程,  可以有不同版本同時進行

Centralized VCS (ex. cvs, svn)

GitHub(2008)
1. Git only
2. 用 Ruby on rails 寫的
3. 提供用戶跟蹤其他用戶、組織、軟體庫的動態，對軟體代碼的改動和 bug 提出評論
4. 提供了圖表功能，用於顯示開發者們怎樣在代碼庫上工作以及軟體的開發活躍程度

Setup
git config --global user.name “Bill”
git config --global user.email bill@gmail.com
git config --global color.ui true
git config –global core.editor vim
(save in ~/.gitconfig)

Create your own gh-pages

Pull request
請求其他人code review, 幫助完成新feature
程式碼可以在獨立的 Branch 運作
過程記錄詳細且完整
夥伴們參與更容易

Flow :
     1. Create branch
     2. Start pull request
     3. Discuss , coding and commit
     4. Merge back to master
